Fumblestyle

I'm fumbling around with stylesheets using Sass (SCSS) with Koala.

Introduction

This is a learning project to create a simple CSS stylesheet that can be used as a starting point for future projects. The focus is on simplicity, whilst making sure that the information looks good on both large and small screens. I've put this on Github too and welcome any feedback/criticism of these fumblings!

CSS Resources

I'll link to tools, tutorials and relevant articles as I fumble along. The smily-face logo utilises the 'fontelico' font via weloveiconfonts.com and I used Adobe's colour wheel tool to pick the colour scheme. A lot of help came from CSS Tricks and W3 Schools was a handy reference.

Sections

Each of these sections is wrapped in a div with max-width: 960px so that I could set the background colour across the full width of the screen but keep the content to 960px. I thought about implementing this example of full browser width bars, but it felt like overkill for the sake of avoiding one wrapper div (per section...) I might return to it in future.

Images

I have set img { width: 100%; } as a default for responsive images. I imagine there will be more to it than this, but it should serve as a starting point so that if images are dropped into the page they don't look too bad on whatever screen size.

Thanks to placekitten.com for providing kitten placeholder images in any dimensions. I also like placehold.it for sites where a lower cuteness factor is more appropriate.

Simple Layout

This is intended to be a simple structure for creating rows and columns.

Here is a single row with three columns which I gave the class name .col-33. The '33' represents the approximate width of the column depending on how many you want in a row. More classes have been added to allow for different layouts: .col-25, .col-75, .col-66 and so on.

Block 1 has just one sentence in it.
Block 2 has more than one sentence. In fact it has multiple sentences because we want this to be the tallest block, height-wise. Hopefully this will do the trick.
Block 3 has two sentences. It should be taller than block 1 but not block 3.

The .col-xx divs are wrapped in a div with .row { font-size: 0; } class; one of the solutions from CSS-tricks for how to fight unwanted gaps between inline-block elements.

Responsive-ish

The .col-xx divs are set to display: inline-block with a media query changing this back to display: block for smaller screens. Meanwhile, here are more kittens to illustrate different column widths (note the image heights have been calculated to be equal - the layout does not do this automatically.)

A rant about CSS layouts

I am perplexed by how difficult it is to put text in boxes and place those boxes exactly how you want them. I know there are frameworks and grid patterns and so on and so forth. But it is remarkably complicated considering the task at hand. And I'm not trying to be dismissive of CSS; it's just that if I spend half an hour to place these three divs neatly in a line and turn to my wife to share this triumph, you can't blame her for looking at me like "wtf..?"
