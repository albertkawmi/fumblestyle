Introduction

I find the idea of styleguides quite interesting. I am not a front-end developer, but this seems like the best way to start any serious project; create a single HTML page that contains all the different elements and design patterns that you'll use in a project. A thoroughly put-together styleguide allows you to see what impact your CSS changes have across all your design elements, saving you having to test against everything. (I guess...)

The smily-face logo utilises the 'fontelico' font via http://weloveiconfonts.com/

Boxes in a row...

A very simply way of lining up boxes of content will suffice for most simple websites. These are divs are set to display: inline-block; with a media query to switch to block representation for smaller screens. The width is set as a percentage with margin also as a percentage.

Block 1 has just one sentence in it.
Block 2 has more than one sentence. In fact it has multiple sentences because we want this to be the tallest block, height-wise. Hopefully this will do the trick.
Block 3 has two sentences. It should be taller than block 1 but not block 3.

As a newcomer to web design, I have to say I am genuinely perplexed by how difficult it is to put text in boxes and place those boxes exactly how you want them. I know there are frameworks and grid patterns and so on and so forth. But it is remarkably complicated considering the task at hand. And I'm not trying to be dismissive of CSS; it's just that if I spend half an hour to place these three divs neatly in a line and turn to my wife to share this triumph, you can't blame her for looking at me like "wtf..?"

Images

I have set img { width: 100%; } as a default for responsive images. I imagine there will be more to it than this, but it should serve as a starting point so that if images are dropped into the page they don't look too bad on whatever screen size.

I love placehold.it - a great service for dropping sizable placeholder images into your designs. But placekitten.com is better if you need kittens.
